Small Cloud Backup

When we choose to use cloud services to support our new web application we choose the best ones that fit specific purpose. Which source control provider? Which cloud storage? Which CRM? Which project management tool?

And as we choose the best our own application and our business starts to depend on them.

Now each and every cloud service can have its own protection from various disasters. And it starts to spread.

SCB is an answer to a cloud application backup.

Design

SCB is in itself a cloud application as well. It is designed to be run on Heroku but can also be run on other aaS as well as on your own dedicated servers.

It provides three main components:

  • The backuper that can connect to specific services and fetch data, prepare the backups and store it somewhere,
  • The scheduler that can queue the backup tasks,
  • The config - a configuration application.

By design you own the instance of this application. You can deploy a copy of it on Heroku and it will be private to you. No data will (or, for that matter, can) be shared between two instances of SCB.
